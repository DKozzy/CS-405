# CS-405

Welcome to the CS-405 repository! This repository contains projects and assignments completed as part of Southern New Hampshire University's **CS-405: Secure Coding** course. Throughout this course, I learned to identify, analyze, and mitigate software vulnerabilities while implementing secure coding practices in C++. The assignments in this repository demonstrate the application of defensive programming techniques, secure software development principles, and vulnerability remediation using both manual code review and static analysis tools.

---

# About the Course

CS-405 focuses on building secure software by identifying common vulnerabilities and implementing techniques to prevent them. Throughout the course, I worked with legacy C++ code, improved software reliability, and applied industry best practices for secure software development.

Key topics covered include:

- Secure software development lifecycle
- Defensive programming techniques
- Buffer overflow prevention
- Numeric overflow and underflow protection
- Exception handling
- SQL injection mitigation
- Encryption and secure file handling
- Static code analysis
- Security policy development
- Secure coding standards

## Programming Activities

### Numeric Overflow

Introduced overflow and underflow protection using `std::optional` and boundary checking with `std::numeric_limits` to prevent unsafe arithmetic operations.

### Buffer Overflow

Analyzed legacy C++ code to identify unsafe memory operations and implemented safer programming practices to eliminate potential buffer overflow vulnerabilities.

### SQL Injection Fix

Added validation logic to detect common SQL injection patterns before execution, preventing malicious user input from compromising database queries.

### Encryption Activity

Implemented file encryption and decryption while demonstrating secure file handling and proper exception management during input/output operations.

### Exceptions Activity

Improved application reliability by replacing unsafe error handling with structured C++ exception handling and custom exception classes.

### Static Code Analysis

Used automated analysis tools to identify potential software defects, compare findings between analysis tools, and evaluate how static analysis improves software quality.

# Why These Projects Matter

Software security is no longer an optional feature—it's a fundamental part of software development. Throughout this course, I learned that writing functional code is only part of creating quality software. Developers must also anticipate how software can be misused or attacked and design applications that remain secure under unexpected conditions.

These projects demonstrate practical methods for identifying vulnerabilities, improving software reliability, and implementing defensive programming techniques. By analyzing insecure code and applying modern C++ security practices, I developed a stronger understanding of how secure software is designed, tested, and maintained.

---

# Future Applications

The concepts learned throughout CS-405 extend far beyond these assignments and can be applied to virtually any software development project. Moving forward, I plan to continue incorporating secure coding practices into future work by:

- Validating user input before processing
- Writing code that safely handles unexpected conditions
- Performing regular static code analysis
- Following secure software development best practices
- Designing software with maintainability and security in mind
- Continuously evaluating applications for potential vulnerabilities

The skills developed throughout this course provide a strong foundation for building software that is not only functional, but also secure, reliable, and maintainable.
